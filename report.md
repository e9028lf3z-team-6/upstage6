# Frontend 변경 안내 (미적용)

이번 수정에서는 프론트엔드 코드를 변경하지 않았습니다. 다만 백엔드에 로그인/제한 모드가 추가되었으므로, 아래 변경이 필요할 수 있습니다.

## 필요 변경 사항
- 로그인 URL을 하드코딩(`http://localhost:8000/api/auth/login`)하지 말고 `VITE_API_BASE` 기반으로 동적으로 구성해야 합니다.
- 로그인 상태에 따라 분석 모드가 달라지므로, UI에 "제한 모드(개연성만)" 안내 배지를 표시하는 편이 좋습니다.
- 401 응답 시 토큰 삭제 외에 사용자 알림/재로그인 유도 메시지가 있으면 UX가 개선됩니다.

## 영향 범위
- `frontend/src/App.jsx`
- `frontend/src/api.js`

## 비고
백엔드에서 `/api/auth/*`, `/api/analysis/run/{doc_id}` 동작이 변경되어, 배포 환경에서는 프론트가 환경 변수 기반으로 로그인 URL을 구성해야 정상 동작합니다.
